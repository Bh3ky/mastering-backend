## Object-Oriented Programming

**OBJECTIVES**

1. Purpose of Object-Oriented Programming
    - understand why OOP exists
    - be able to explain:
        - modularity
        - code reuse
        - abstraction
        - maintainability
        - scalability
    - procedural vs object-oriented design

2. Classes and Objects
    - defining and instantiating classes
        - create classes using `class`
        - instantiate objects
        - understand:
            - class blueprint
            - object instance
            - object state
            - object behavior

    - working with Attributes
        - define instance attribute
        - define class attributes
        - distinguish between:
            - mutable vs immutable attributes
            - shared vs per-instance state

    - using methods correctly
        - instance methods
        - class methods
        - static methods
        - understand:
            - `self`
            - `cls`
            - `method binding`

3. Constructors and Object Lifecycle
    - understanding constructors
        - use `__init__`
        - initialize object state
        - validate constructor inputs
        - understand:
            - default parameters
            - optional attributes
            - initialization patterns
    
    - understanding object representation
        - implement:
            - `__str__`
            - `__repr__`
        - know the difference between:
            - developer-facing representation
            - user-facing representation

4. Encapsulation
    - applying encapsulation principles
        - public attributes
        - protected conventions (`_attributes`)
        - private name mangling (`__attributes`)
    - understand:
        - controlled access
        - hiding implementation details
    - using getters, setters, and properties
        - implement computed attributes
        - use `@property`
        - use setters safely

5. Inheritance
    - implementing inheritance
        - crete parent and child classes
        - reuse functionality
        - extend behavoir
        - understand:  
            - `super()`
            - method overiding
            - method resoluton order (MRO)

    - differentiating inheritance types
        - single inheritance
        - multiple inheritance
        - multilevel inheritance
        - understand risk of:
            - tight coupling
            - diamond inheritance problem

6. Polymorphism
    - applying polymorphism
        - same interface, different implementations
        - method overiding
        - duck typing

7. Abstraction
    - use abstract base classes
        - use Python's abc module
        - create abstract methods
        - enforce contracts/interfaces

8. Composition vs inheritance

9. Dataclasses
